# 1. LLM(Large Language Model)이란?

## LLM의 본질

> **LLM은 '다음 토큰(Next Token)을 확률적으로 예측하는 모델'이다.**

사람처럼 문장을 이해하는 것이 아니라, 지금까지 입력된 문맥을 바탕으로 **가장 확률이 높은 다음 토큰을 계속 생성**하여 문장을 만든다.

```text
입력
대한민국의 수도는

↓

예측

서울
```

즉, LLM은 **언어의 패턴을 학습한 확률 모델**이다.

---

## Token이란?

토큰(Token)은 AI가 문장을 처리하는 최소 단위이다.

예시

```text
안녕하세요.

↓

안녕 / 하세요 / .
```

토큰 수는 언어마다 다르며 일반적으로 한국어는 영어보다 더 많은 토큰을 사용한다.

> API 사용 비용 역시 토큰 수를 기준으로 계산된다.

---

# 2. LLM의 학습 과정

LLM은 크게 3단계를 거쳐 만들어진다.

| 단계 | 이름 | 설명 |
|------|------|------|
| 1 | Pre-training | 인터넷 규모의 데이터를 학습하여 언어 패턴과 지식을 습득 |
| 2 | SFT (Supervised Fine-Tuning) | 좋은 질문과 답변 데이터를 학습하여 사용자의 지시를 따르도록 개선 |
| 3 | RLHF | 사람의 피드백을 이용해 선호되는 답변 스타일을 학습 |

### 비유

```text
Pre-training
↓
도서관의 모든 책을 읽는다.

SFT
↓
선배에게 업무를 배운다.

RLHF
↓
고객 피드백을 받아 더 좋은 응대를 배운다.
```

# 3. LLM의 한계

## 1. Hallucination (환각)

존재하지 않는 사실도 그럴듯하게 생성하는 현상이다.

LLM은 모르는 내용을 추측해서 답하려는 경향이 있다.

### 해결 방법

- RAG
- 검색 API
- 공식 문서 참조
- 출처 제공

---

## 2. Knowledge Cutoff

LLM은 학습이 끝난 이후의 최신 정보를 알지 못한다.

예시

- 오늘 뉴스
- 최신 주가
- 최신 라이브러리 버전

### 해결 방법

- Web Search
- 외부 API
- 데이터베이스 조회

---

## 3. Context Window

한 번의 대화에서 기억할 수 있는 최대 토큰 수이다.

대화가 길어질수록 앞부분의 정보를 잊어버릴 수 있다.

### 해결 방법

- 중요한 내용 요약
- 새로운 대화 시작
- RAG 활용

---

## 4. Non-determinism (비결정성)

같은 질문을 여러 번 해도 결과가 조금씩 달라질 수 있다.

이는 확률적으로 토큰을 선택하기 때문이다.

### 해결 방법

- Temperature 낮추기
- 출력 형식 고정
- JSON 출력

# 5. 모델 선택 기준

모델은 성능만 보고 선택하지 않는다.

## 비용과 성능의 균형

```text
비용 ↑
성능 ↑

비용 ↓
성능 ↓
```

예를 들어,

- 간단한 요약
- 대량 분류
- 번역

같은 작업은 저렴한 모델이 효율적이다.

반면

- 복잡한 추론
- 전략 분석
- 긴 문서 분석

은 고성능 모델이 적합하다.

와 같은 하이브리드 구성이 많이 사용된다.

# 9. 프롬프트 엔지니어링

프롬프트 엔지니어링은 **모델에게 원하는 결과를 얻기 위한 작업 지시서를 작성하는 기술**이다.

좋은 프롬프트일수록 결과의 품질이 높아진다.

---

## 프롬프트의 4요소

### 1. Role (역할)

누구의 입장에서 답변할 것인가?

```text
너는 10년차 백엔드 개발자다.
```

---

### 2. Context (맥락)

현재 상황과 대상은 누구인가?

```text
신입 개발자 교육 자료이다.
```

---

### 3. Task (작업)

무엇을 수행해야 하는가?

```text
DDD를 쉽게 설명해라.
```

---

### 4. Format (형식)

어떤 형태로 출력할 것인가?

```text
Markdown

표 포함

예시 포함

3줄 요약
```

---

## 프롬프트 예시

```text
Role
너는 금융회사 백엔드 개발자다.

Context
신입 교육 자료를 작성한다.

Task
LLM의 한계를 설명한다.

Format
Markdown 형식
표 포함
예시 포함
```

---

# 10. 프롬프트 작성 팁

## 1. 구분자 사용

명령과 데이터를 명확하게 구분한다.

예시

```text
#

---

[]
```

---

## 2. 작업을 단계별로 분리

```text
1. 정보 추출

↓

2. 분류

↓

3. 요약
```

복잡한 작업일수록 품질이 향상된다.

---

## 3. 출력 예시 제공

원하는 결과 형태를 미리 보여주면 형식을 잘 지킨다.

이는 Few-shot Prompting의 기본 개념이다.

# 11. AI 업무 자동화 단계

생성형 AI 활용은 다음과 같이 발전한다.

| 단계 | 설명 |
|------|------|
| Lv.1 | 채팅으로 직접 활용 |
| Lv.2 | GPTs, Projects 등 반복 작업 구성 |
| Lv.3 | Workflow(n8n 등) 자동화 |
| Lv.4 | AI Agent가 목표를 달성하도록 수행 |

궁극적으로는

```text
사람이 직접 수행

↓

워크플로우 자동화

↓

AI Agent가 스스로 계획하고 실행
```

하는 방향으로 발전한다.
